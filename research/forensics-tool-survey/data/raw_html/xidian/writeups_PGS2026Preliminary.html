        h1 = hashlib.sha256((p + SALT).encode()).hexdigest()

    salt = base64.b64decode(data["salt"])
    master_key_blob = base64.b64decode(data["masterKey"])
    iterations = data["iterations"]

    # Parse opdata1 format
    HEADER_LENGTH = 8
    HMAC_LENGTH = 32

    if master_key_blob[:HEADER_LENGTH] != b"opdata01":
        raise ValueError("masterKey is not in opdata1 format")

    expected_hmac = master_key_blob[-HMAC_LENGTH:]
    hmac_d_data = master_key_blob[:-HMAC_LENGTH]

    VIN 码

    汽车 VIN 码, 全称 Vehicle Identification Number, 译为车辆识别码, 常称作"车架号". 每个 VIN 码对应唯一的一辆车.

    VIN 码由 17 位字符组成, 包含数字和字母. 为了避免混淆, 英文字母I、O、Q不使用. VIN 码的组成为:

    • 3 位 WMI (World Manufacturer Identifier, 世界制造厂识别代号): 我国常见的 WMI 有
    • LSG: 上汽通用
    • LSV: 上汽大众
    • LFV: 一汽大众
    • LHG: 广州本田
    • LVS: 长安福特
    • LS5: 长安汽车
    • LHB: 北汽福田
    • LC0: 比亚迪新能源
    • 6 位 VDS (Vehicle Descriptor Section, 车辆说明部分): 说明车辆的一般特性.
    • 8 位 VIS (Vehicle Indicator Section, 车辆指示部分): 车辆制造商为了区别不同车辆而指定的编码, 第 1 位字符代表生产年份.

    例如, 本检材中, 可以在 system/etc/build.prop 中看到 ro.vehicle.vin=LSGXE53W7PS012345, 其中的 LSG XE53W7 PS012345 即为检材来源车辆的 VIN 码. 通过 VIN 码可以得知, 这是一辆由上汽通用于 2023 年生产的汽车.

    T-Box

    OBD

    OBD (On-Board Diagnostics, 车载诊断数据) 是指车辆的自我诊断和报告能力. 现代车载诊断器使用标准化数字通信端口来提供实时数据和诊断故障代码, 从而快速识别车辆内部的故障.

    CAN Bus

    CAN (Controller Area Network, 控制器区域网络) 是一种车辆接口标准, 主要实现 EC (电子控制单元) 间的高效通信, 也被用于各种其他环境中, 例如 3D 打印.

    CAN 是一种广播协议, 通过"仲裁"过程确保数据完整性和优先级: 如果多个设备试图同时发送数据，则允许最高优先级的设备继续传输, 其他设备后退. CAN 协议的常见版本有 CAN 2.0、CAN FD 和 CAN XL.

    DLT

    DLT (Diagnostic Log and Trace, 系统诊断日志) 是 AUTOSAR (AUTomotive Open System ARchitecture, 汽车开放系统架构) 标准的诊断日志协议, 记录 ECU (Electronic Control Unit, 电子控制单元) 级别的关键事件.

    ADAS

    ADAS (Advanced Driver-Assistance System, 高级驾驶员辅助系统) 是协助驾驶员安全操作车辆的技术. ADAS 使用传感器和摄像头等自动化技术来检测附近的障碍物或驾驶员错误并做出相应响应, 实现不同级别的辅助驾驶或自动驾驶.

    在 OBD 数据 obd\history\freeze_frame_B0070.bin 中可以看到以下数据:

    Text Only

    根据 SAE J1979 标准定义的 OBD-II PID, 前 2 字节 0xB007 为 OBD 代码, 代表 Driver Seatbelt Pretensioner "A" Deployment Control (Subfault) (驾驶员安全带预紧器展开控制故障)错误, 参考 obd-codes - b0070.

    OBD 代码的第一位代表错误发生的组件, 不同组件的 OBD-II 数据格式略有不同:

    • P: 动力系统(发动机和变速器等)
    • B: 车身(安全气囊/SAS, 安全带, 空调, 舒适性和内部电子设备等)
    • C: 底盘(ABS, 牵引, 转向, 悬挂和轮速传感器等)
    • U: 网络(模块通信, CAN 和电力网络等)

    接下来的 0x0D 为 OBD-II 的 PID, 提示接下来的数据为 Vehicle speed (车速). 随后的 0x01 代表数据有 1 字节长. 再随后的 0xB4 为数据, 根据 OBD-II 的定义, 该数值 0xB4 = 180 km/h 即为车速.

    Text Only

    接下来的 0x02 是故障码状态位, 意为"当前存在故障".

    设: 真实 XOR key 长度为 \(K\), 当前猜测 key 长度为 \(k=8\)

    \[ P | \gcd(240, 80) \Rightarrow P | 80 \]

    \[ \mathrm{lcd}(K,4) ∣ 80 \]

    由于:

    \[ 80=2^4\times5 \]

    \[ P\in \{4,8,16,20,40,80\} \]

    \[ K\in\{4,5,8,10,16,20,40,80\} \]

    \[ K \gt k \]

    \[ K\in\{10,16,20,40,80\} \]

    result = []
    for i, c in enumerate(data):

    step4 = step3.encode('utf-8').hex()
